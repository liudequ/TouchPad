# BLE 三设备切换实现思路（触摸板）

## 目标
- 支持在 3 台已配对设备之间快速切换连接。
- 任意时刻仅保持 1 条 BLE HID 连接（避免多连接带来的带宽与复杂度问题）。
- 连接切换后尽快恢复输入可用。

## 推荐方案
采用“3 槽位配对 + 单连接快速切换”：

1. 固件维护 `slot1/slot2/slot3` 三个槽位。
2. 当前连接绑定到一个 `activeSlot`。
3. 用户触发切换后：
   - 断开当前连接；
   - 进入目标槽位重连流程；
   - 成功后更新当前状态。

## 状态机建议
1. `ACTIVE(slotN)`：正常工作状态。
2. `SWITCHING`：收到切换请求。
3. `DISCONNECT_CURRENT`：主动断开当前主机。
4. `RECONNECT_SLOT_N`：按目标槽位进行广播/重连。
5. `TIMEOUT_FALLBACK`：超时后回退（通用广播或返回上一个槽位）。

## 关键实现点
1. 槽位管理
- 增加配置项：`activeSlot(1..3)`、每槽位是否已绑定。
- 复用现有 `InternalFS` 进行持久化。

2. 切换触发
- 优先使用串口命令验证：`SLOT 1|2|3`。
- 稳定后再绑定手势（例如四指左右滑循环切槽）。

3. 连接策略
- 切槽时先断开当前连接。
- 启动目标槽位重连窗口（建议 8~15 秒快速广播）。
- 目标槽位未绑定时进入该槽位配对模式。

4. 绑定与清理
- 支持按槽位管理配对：`PAIR SLOT n`、`UNPAIR SLOT n`。
- 保留总清除：`PAIRCLR`。

5. 状态反馈
- LED 区分槽位状态（例如不同闪烁节奏表示 slot1/2/3）。
- 串口 `GET` 增加状态输出：`activeSlot`、`slot1/2/3 bonded`、`connectedSlot`。

## 落地顺序（建议）
1. 先实现串口切槽与状态查询（不改手势）。
2. 再接入槽位绑定状态持久化。
3. 最后增加手势触发切槽。

## 当前项目中的最小可用版本（MVP）
第一阶段仅做：
- `SLOT n` 切换命令；
- 当前槽位状态查询；
- LED/串口可见反馈；
- 保持现有 HID 主逻辑不变。
