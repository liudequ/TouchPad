# 稳定性与防抖优化清单（待逐步实施）

目标：提升触摸板手感稳定性，降低抖动/漂移，保证输出节拍一致。

## 1) 输入层：坐标低通滤波
- 对原始坐标做一阶 IIR 低通（如 `x_f = x_f + a*(x - x_f)`）。
- 作用：压掉传感器噪声，减少后续算法的抖动放大。
- 建议先在单指路径落地，验证后再应用到双指/三指。

## 2) 运动层：速度/加速限幅
- 在 `MAX_DELTA` 基础上增加速度上限或加速度变化限幅。
- 作用：防止偶发尖峰导致光标跳变。

## 3) 手势层：模式切换滞回
- 单指/双指/三指切换引入 1–2 帧滞回或时间阈值。
- 作用：避免在边缘状态来回抖动导致误切换。

## 4) 输出层：累积量上限
- 对 `accumX/Y`、`accumScroll` 设置上限，避免长时间抖动后“突然跳”。
- 作用：限制最坏情况位移，改善主观稳定性。

## 5) 滚动微动抑制
- 对 `scrollVel` 加静止阈值，小于阈值直接归零。
- 作用：消除细微抖动引起的“慢滚”。

## 6) I2C 读异常恢复
- 读失败累计次数后重置 I2C 或重新初始化触摸板。
- 作用：避免读卡死导致节拍抖动或输入断流。

## 实施顺序建议
1) 输入层低通滤波  
2) 输出层累积上限  
3) 滚动微动抑制  
4) 模式切换滞回  
5) 速度/加速限幅  
6) I2C 异常恢复

---

## 轨迹贴近真实手指路径的做法
- 关闭或弱化加速：`accelFactor = 0`，`maxAccel = 1` 或小幅度加速。
- 降低/关闭死区：`MOVE_DEADBAND` 尽量小。
- 合理放宽限幅：`MAX_DELTA` 不要过低，避免“断笔感”。
- 提高输出频率：从 60 Hz 提升到 90/125 Hz（需评估功耗与 BLE 能力）。
- 低通滤波系数调小：更平滑但会引入轻微延迟，需权衡。

## 与稳定性优化的冲突点
- 加速（稳定手感）会扭曲慢速轨迹形状，影响“画圆像圆”。
- 死区（抑抖）会让小幅度运动变成分段，产生棱角。
- 过强限幅会导致轨迹断裂或变成多段直线。
